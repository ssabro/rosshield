## Added

### SwappableSigner + KeyRotator + quarterly cron (Stage 10.D-3+4)

`internal/platform/signer/swappable.go::SwappableSigner` — RWMutex queue 패턴 hot-swap wrapper. Sign 호출은 RLock 으로 직렬화 (동시 다수 OK, in-flight 시 Swap 은 모든 RLock 해제까지 대기 — D-P10D-3 Queue 결정 일관). `internal/domain/audit/keyrotation/rotator.go::KeyRotator` 신규 — 단일 storage.Tx 안에서 audit_chain_keys append + 이전 epoch revoke + audit.chain.key_rotated emit + 신호 후 SwappableSigner.Swap. self-sign + Verify round-trip fail-safe. `internal/platform/scheduler/keyrotationjob/keyrotationjob.go` — quarterly cron job 어댑터 + DefaultQuarterlySpec `@every 2160h`. HA 활성 시 cronsched RoleProvider gate(E25 Stage 4a) + KeyRotator 내부 leader gate(2 단계 defense-in-depth, Stage 10.D-6 lazy SetLeader 결선).

### 마이그레이션 0037 + 0038 (Stage 10.D-2 + 10.D-3)

- `migrations/sqlite/0037_audit_chain_keys.up.sql` + `migrations/postgres/0037_audit_chain_keys.up.sql` — `audit_chain_keys` 테이블 (tenant_id + epoch + key_id + public_key_hex + keystore_handle + created_at + revoked_at + created_by + audit_entry_seq). bootstrap epoch=1 row 자동 INSERT (placeholder, 첫 부팅 시 갱신).
- `migrations/sqlite/0038_audit_entries_key_epoch.up.sql` + `migrations/postgres/0038_audit_entries_key_epoch.up.sql` — `audit_entries.key_epoch INTEGER NULL` 컬럼 추가. NULL = legacy entry (v0.9.0 이하), 채워진 값 = 본 entry INSERT 시점 SwappableSigner epoch.

### fg-verify v2 — epoch별 public key 검증 (Stage 10.D-5)

`cmd/rosshield-audit-verify/rotation.go` 신규 + `export_verify.go` 확장. bundle 안에 `chainKeyEpochs` 메타로 epoch 별 public key 직렬화. 검증 시 `entry.key_epoch` 가 채워진 entry 는 해당 epoch 의 public key 로, NULL entry 는 v0.9.0 backward compat 모드 (단일 head public key) 로 verify. v0.9.0 이하 bundle 은 chainKeyEpochs 누락 → 자동 v1 mode fallback.

### Emergency override CLI + admin endpoint (Stage 10.D-6)

- `POST /api/v1/audit/rotation/abort` — `internal/api/handlers/audit_rotation.go` 신규. admin 권한(`RequirePermission(ResourceTenantAdmin, ActionAdmin)`). body `{"reason":"<string>"}` (optional). 응답 `{"aborted":true, "auditEntryId":<seq>, "abortedAt":"<iso8601>", "previousEpoch":<n>, "reason":"<echo>"}`. KeyRotator 미주입 시 503.
- `rosshield audit rotation abort --reason "<text>"` — `cmd/rosshield/audit.go` 신규. table/json 출력, exit code R11-8 매핑 (4xx→2, 5xx→3).
- `KeyRotator.Abort(ctx, reason, actor)` — atomic abort flag set + audit.chain.rotation_aborted event emit. Idempotent — 진행 중 rotation 없을 때도 emit (다음 rotation 1 회 차단). RotateNow critical 지점(진입 / Allocator 후 / Tx 직전) 3 곳에서 abort flag polling.
- `KeyRotator.SetLeader(LeaderProvider)` — bootstrap HA Manager 결선 시점 lazy 주입 (defense-in-depth leader-side 2 단계 활성). cmd/rosshield-server/bootstrap.go 의 e25 + patroni 분기 모두에 호출 추가.

### audit.chain.key_rotated + audit.chain.rotation_aborted events

`audit.chain.key_rotated` (Stage 10.D-3) — 매 rotation 마다 swap 직전 entry. payload `{"fromEpoch":N,"toEpoch":N+1,"newKeyId":"key_...","publicKeyHex":"<hex>","trigger":"scheduler|manual|cli"}`. `audit.chain.rotation_aborted` (Stage 10.D-6) — Abort 호출마다 entry. payload `{"reason":"<text>","actor":"<user>","previousEpoch":N}`.

### Metrics: rosshield_audit_rotation_total + rosshield_audit_key_epoch (Stage 10.D-3)

- `rosshield_audit_rotation_total{status="success|failed|skipped"}` Counter — rotation 호출 결과 누적.
- `rosshield_audit_key_epoch{tenant_id}` Gauge — tenant 별 현재 활성 epoch.

### Ops docs (Stage 10.D-7)

`docs/operations/audit-chain-key-rotation.md` 신규 — §1 개요 + §2 hot-swap timeline + §3 모니터링 (Prometheus + Grafana panel + check-health) + §4 정상 절차 (운영자 개입 0) + §5 emergency override 절차 + §6 broken rotation 대응 + §7 외부 감사 호환성 + §8 채널 정책 + §9 한계 + §10 참조.

## Changed

### bootstrap signer 경로 (Stage 10.D-3+4)

`cmd/rosshield-server/bootstrap.go` — audit chain signer 가 raw `signer.Signer` → `*signer.SwappableSigner` 로 wrap. 기존 audit checkpoint job + audit emit 경로는 SwappableSigner.Sign 인터페이스 호환 유지 (Signer interface 그대로 implement). 호출자 변경 없음 — 변경은 wrapper 합성 1 곳뿐. HA Manager 결선 시점에 `keyRotator.SetLeader(haMgr|patroniRP)` lazy 호출 추가 (Stage 10.D-6).

## Migration Notes

### Customer 영향

- v0.9.0 customer 는 v0.10.0 binary update + 마이그레이션 0037/0038 자동 적용 후 정상 동작.
- 자동 quarterly rotation 이 시작되지만 운영자 알람 발생 안 함 (INFO 로그 + Prometheus counter 만). 첫 rotation 은 `ROSSHIELD_AUDIT_CHAIN_KEY_ROTATION_SCHEDULE` 환경 변수 설정 시점부터 90 일 후 (또는 cron tick 도달 시).
- 자동 rotation 비활성 유지를 원하면 `ROSSHIELD_AUDIT_CHAIN_KEY_ROTATION_SCHEDULE=""` 명시 (default = 빈 값 = 비활성). quarterly 활성화는 customer 옵트인.
- 외부 감사인이 v0.9.0 이하 bundle 을 받았다면 fg-verify v1 mode 자동 동작 (chainKeyEpochs 누락 → 기존 단일 head public key 검증). v0.10.0 이후 bundle 은 v2 epoch별 검증.

### Prometheus / Grafana

- 신규 metric 2 종 (`rosshield_audit_rotation_total` + `rosshield_audit_key_epoch`) 활성화는 자동 — Prometheus scrape target 이 이미 설정되어 있으면 즉시 노출.
- Grafana panel 추가 권장 — `docs/operations/audit-chain-key-rotation.md` §3.2 예시 query 참조. 본 round 는 별도 dashboard JSON 미작성 (별 epic 위임 — `deploy/grafana/rosshield-dashboard.json` 갱신 시점에 통합).

## 한계 (carryover)

### v0.10.0 신규 carryover

- **audit hash chain input 미포함**: `key_epoch` + `leader_epoch` 는 `audit_entries` 컬럼으로만 보존. hash chain 자체 input 에 포함되지 않음 (외부 검증 도구 호환성 유지). 향후 SC-12 강화 시 별 epic.
- **bootstrap epoch=1 placeholder**: 0037 마이그레이션이 bootstrap row 를 INSERT 할 때 keystore handle 은 빈 문자열 / public key 는 placeholder. 첫 부팅 시 실제 signer 정보로 자동 갱신.
- **manual rotation endpoint 미구현**: emergency abort 만 노출. 즉시 rotation 트리거가 필요하면 quarterly schedule 을 일시 단축 후 restart.
- **Grafana dashboard panel 별 작성 위임**: `deploy/grafana/rosshield-dashboard.json` 갱신은 본 round 미진행 — 다음 dashboard 통합 round.
- **multi-tenant epoch 분리 미구현**: 본 round 는 system tenant 단일 인스턴스 가정. customer 도메인 tenant 별 epoch 는 별 round.
