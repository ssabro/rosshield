## Fixed

### snap `check-health` hook 30s hard timeout 회피 (`96305be`)

snapd의 check-health hook은 30s hard timeout이 강제됩니다(v0.8.4 첫 실 검증에서 정확치 확정). 기존 hook이 max 30s polling을 그대로 사용해 정확히 한도 hit → `snap install` step 자체가 fail.

**v0.8.4 Snap Smoke 실패 로그**:
```
error: cannot perform the following tasks:
- Run health check of "rosshield" snap (run hook "check-health": <exceeded maximum runtime of 30s>)
```

**hook 변경**:
- `CHECK_TIMEOUT=30` → `CHECK_TIMEOUT=10` (snapd 30s hard timeout 안에 안전 margin).
- polling interval 2s → 1s, curl `--max-time 5` → `--max-time 3`.
- daemon ready 안 됐으면 빠르게 `waiting`으로 종료 — snapd 주기 재호출(약 5분) 또는 외부 trigger에서 catch up.

### snap-smoke CI 외부 hook trigger 패턴 (`96305be`)

daemon 막 부팅 중 hook이 waiting으로 종료한 경우를 cover:

- `snap health — trigger + verify check-health` (첫 install 후, step 이름 변경).
- `snap health — trigger + verify after refresh` (refresh round-trip 후, step 이름 변경).

매 polling iteration에서 `sudo snap run --hook=check-health rosshield`를 직접 호출하여 hook 재실행 → status okay 전환 시 break. 30s polling 동안 hook 30회 재실행 + status 확인. 실패 시 snap logs까지 출력.
