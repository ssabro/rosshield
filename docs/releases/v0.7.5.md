## Added

### Phase 8 Stage 7 testcontainers e2e fixture + MR.T1·T7 (`3803369`)

`internal/platform/storage/postgres/replication_integration_test.go` (build tag `integration`).

#### fixture (`newReplicationFixture`)

- testcontainers Docker network에 2 PG container 자동 spawn
- 양쪽 `wal_level=logical` + `max_wal_senders=10` + `max_replication_slots=10`
- primary alias `primary-pg`, standby alias `standby-pg` (network-internal DNS)
- 양쪽 Migrate + PUBLICATION/SUBSCRIPTION 자동 setup

#### MR.T1 — TestReplicationLagWithin1Second

- primary INSERT → standby polling 100ms → assert lag < 1s
- RPO ≤ 1분 목표 cover

#### MR.T7 — TestTenantMetaReplicated

- primary tenant CREATE → standby에서 slug/display_name/status 정확 검증
- R5 cross-region tenant 격리 cover

### Phase 8 Stage 7 MR.T4·T5·T6·T8 추가 (`d842583`)

#### MR.T4 — TestFailoverPromotesStandby

- ALTER SUBSCRIPTION DISABLE → standby write 수용 + primary와 isolation
- application-level leader-election restart는 별 layer carryover

#### MR.T5 — TestAuditChainHeadSHACrossRegion

- primary audit entry 5건 INSERT + audit_chain_heads upsert
- standby에서 head_hash 일치 + audit_entries count 검증
- R2 cross-region audit chain 일관성 cover

#### MR.T6 — TestLeaderEpochSchemaPropagates

- audit_entries.leader_epoch column(0023 마이그레이션) 정확 replicate
- value=42로 INSERT → standby에서 동일 epoch 조회
- split-brain 방어 base (application fence enforcement는 별 layer)

#### MR.T8 — TestReplicationLagMeasurable

- primary write 후 pg_stat_replication view 조회
- subscriber count + write_lsn IS NOT NULL → lag 측정 가능
- Prometheus metric emit(`rosshield_replication_lag_seconds`)은 별 carryover
