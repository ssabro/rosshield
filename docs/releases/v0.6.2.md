## Added

### ROS2 baseline pack Round 3 (`a914735`)

8/8 카테고리 cover 직후 carryover 6건으로 각 카테고리 깊이 확장. layered defense 입증.

#### C4 binary 무결성 깊이 3건
- **`apt_key_valid`** — ROS2 GPG key 유효 + 만료 ≥ 90일 (env `ROSSHIELD_APT_KEY_FILE` override 가능)
- **`colcon_install_hash`** — `install/setup.bash` sha256 = customer baseline digest 일치 (env `ROSSHIELD_COLCON_BASELINE_SHA256` 미설정 시 PASS skip — false positive 회피)
- **`signed_packages_only`** — `apt-cache policy ros-jazzy-*` 결과 origin 화이트리스트 (packages.ros.org · archive.ubuntu.com · security.ubuntu.com)

#### C5 launch 안전 깊이 3건
- **`param_files_owner`** — launch param yaml owner=ros-user + perms ≤ 0644 (env `ROSSHIELD_ROS_USER` default `ros`)
- **`argv_no_remote_url`** — launch arg에 `http://`/`https://` 외부 URL 없음 (supply chain 공격 차단)
- **`lifecycle_node_used`** — safety-critical 노드는 managed lifecycle 사용 (env `ROSSHIELD_SAFETY_CRITICAL_NODES` 미설정 시 PASS skip)

#### env var 의존 site policy fallback
세 check는 env 미설정 시 PASS skip — false positive 회피. customer onboarding 시 env 정의 절차 안내 필요.

#### paying customer demo 가치
- **supply chain 3 layer 직렬 검증**: apt source → key 만료 → install digest → origin whitelist (ROS2 binary 변조 부재 결정론적 증거)
- **launch 안전 3 layer**: world-writable → owner/perms → remote URL → lifecycle (safety-critical 노드 부팅 안전성)
- enterprise SI 협상 시 22 check 카탈로그 + layered defense 입증
