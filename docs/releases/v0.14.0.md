## Added

### `internal/platform/otel/` SDK + provider scaffold (Stage 11.A-2)

`internal/platform/otel/` 신규 패키지 — Lodestar 전 영역의 단일 tracer provider 진입점:

- `Provider` struct + `NewProvider(ctx, Config)` 생성자 — `Enabled=false` (default) 시 **noop tracer 반환**, exporter 미연결, 호출 비용 거의 0 (R14-1 옵트인 원칙 일관). `Enabled=true` 시 OTLP exporter + parent_based sampler + resource attribute 결선.
- `Config` struct — `Enabled` · `ServiceName` · `ServiceVersion` · `Endpoint` · `ExporterType` (grpc/http, D-P11A-2 both) · `Insecure` (TLS off, dev/air-gap) · `SamplingRatio` (default 0.05) · `Headers` (OTLP auth) · `Region` (multi-region attribute) · `Environment` (deployment.environment attribute).
- `NewSampler(ratio)` — `parent_based(traceidratiobased(ratio))` 권장 default (D-P11A-3). `ratio <= 0` → NeverSample, `>= 1.0` → AlwaysSample, 그 외 → parent_based.
- `newExporter` — gRPC (`otlptracegrpc`) + HTTP (`otlptracehttp`) 양쪽 plug (D-P11A-2 both).
- `newResource` — `service.name` · `service.version` · `rosshield.region` · `deployment.environment` attribute 결선.
- `TraceContext` + `Baggage` composite propagator — W3C standard 표준 일관.
- `Provider.Shutdown(ctx)` — pending span flush + exporter close (default timeout 5s, graceful).
- `NewProviderFromComponents(tp, propagator, enabled)` — in-memory `tracetest.SpanRecorder` 주입 helper (단위 test 결선).
- 단위 test 12 신규 — provider noop short-circuit + endpoint validation + ratio 정규화 + exporter selection + resource attribute + propagator composite + shutdown idempotent + components injection.
- `cmd/rosshield-server/bootstrap.go` 결선 — `buildOtelProvider(ctx, cfg)` 1 helper + `Deps.Otel` field 노출 + provider Shutdown graceful 흐름.

### HTTP middleware — request trace + W3C `traceparent` propagation (Stage 11.A-3)

`internal/api/handlers/otel_middleware.go` 신규:

- `OtelTrace(provider)` middleware — chi router `Use` 결선. incoming HTTP request 마다 root span emit, `traceparent` header 자동 parse (있으면 parent 상속, 없으면 root 신규).
- span name = `HTTP <METHOD> <route>` (chi route pattern 활용) + attribute `http.method` · `http.route` · `http.status_code` · `rosshield.tenant_id` (cross-tenant 분리 필수, R11A-3).
- Enabled=false (noop) 시 middleware 자체 short-circuit — 응답 latency 영향 0.
- 단위 test 8 신규 — incoming `traceparent` parse + root span emit + status_code 부착 + tenant_id 부착 + noop short-circuit + middleware 순서 (`RequireLeaderForWrites` · `StandbyReadOnlyMiddleware` 보다 앞 정렬 검증).

### scan flow 5 span instrument (Stage 11.A-4)

`internal/app/scanrun/scanrun.go` + `internal/app/scanrun/scanrun_span.go` 신규:

- `scan.run` root span (worker entry) — `rosshield.tenant_id` · `rosshield.scan_id` · `rosshield.fleet_id` · `rosshield.pack_id` attribute.
- 5 child span 순차 emit:
  - `scan.ssh.connect` — `rosshield.robot_id` · `ssh.host` attribute. sshpool 재사용 캐시 hit/miss 분석.
  - `scan.check.exec` — `check.id` · `check.category` · `ssh.exit_code` attribute. 특정 check 의 host-side bash 병목 분석.
  - `scan.check.evaluate` — `check.id` · `check.severity` · `check.outcome` attribute. evaluator 분기 latency.
  - `scan.evidence.write` — `evidence.byte_count` · `evidence.format` attribute. DB I/O · blob store 지연 분석.
  - `scan.publish` — `audit.event.action` · `eventbus.topic` attribute. audit emit + eventbus publish hot path.
- worker goroutine ctx propagation 정확성 검증 — `Tracer` 가 nil 인 경우 noop tracer fallback (기존 tester 회귀 0).
- 단위 test 5 신규 (`scanrun_otel_test.go`) — in-memory `tracetest.SpanRecorder` 기반 5 span tree + attribute 검증 + noop short-circuit.

### multi-region + Patroni REST trace propagation + `httpclient.WrapClient` (Stage 11.A-5)

`internal/platform/httpclient/otel.go` 신규 (helper) + `internal/api/handlers/replication.go` + `internal/platform/replication/middleware.go` + Patroni REST adapter 결선:

- `httpclient.WrapClient(client, tp, target)` + `httpclient.WrapTransport(base, tp, target)` — outbound HTTP `traceparent` 자동 inject + `rosshield.outbound.target` attribute (`patroni` · `webhook` · `llm` · `anchor`) 부착. tp == nil 또는 Enabled=false 시 short-circuit (overhead 0).
- multi-region 4 replication endpoint (`heartbeat` · `failover` · `status` · `health`) 의 incoming HTTP span + Patroni REST adapter 의 outbound HTTP span 자동 연결 → cross-region distributed trace 완성.
- standby read-only middleware + `RequireLeaderForWrites` 와 순서 정렬 — otel middleware 가 앞 (401/409 응답도 span emit).
- bootstrap 결선 3 위치 — Patroni REST adapter (`Tracer: otelProvider.Tracer("rosshield/platform/ha/patroni")`) + webhook client (`httpclient.WrapClient(..., "webhook")`) + LLM client adapter (4 provider `httpclient.WrapClient(..., "llm")` 일괄).
- 단위 test 6 신규 (`httpclient/otel_test.go`) — Wrap noop short-circuit + outbound `traceparent` inject + target attribute 부착 + client field 보존 + transport nil fallback.

```bash
# low-volume customer (< 100 scan/day) — 모두 emit.
--otel-sampling=1.0

# high-volume customer (> 10k scan/day) — 1%.
--otel-sampling=0.01

# tail-based sampling (collector 단 정책) 결합 — Lodestar 측 1.0 + collector 단 errors/slow/llm 100%.
--otel-sampling=1.0  # + otel-collector tail_sampling processor 활성 (docs/onboarding/otel-collector-setup.md §8)
```
