## Added

### `/regions` 페이지 + 3 카드 (Stage 10.A-2·3·4)

`web/src/routes/_authenticated/regions.tsx` 신규 (admin 권한 게이트 — `tenant_admin/admin`). Phase 8 PG cross-region replication + Phase 9 Patroni 자동 failover 토폴로지를 한곳에서 노출:

- **RegionHealthCard** (10.A-2) — `web/src/components/regions/RegionHealthCard.tsx` 신규. region별 role(primary/standby/failed) + lag(healthy/warning/delayed) + 엔드포인트 + last_replay/last_heartbeat 시각 표시. replicas 미시드 환경에서 empty state.
- **AuditConsistencyCard** (10.A-3) — `web/src/components/regions/AuditConsistencyCard.tsx` 신규. 현 region의 audit chain head sha + seq + 갱신 시각 표시. 외부 region 비교는 Prometheus 외부 수집기 위임(self-only note).
- **RegionTimelineCard** (10.A-4) — `web/src/components/regions/RegionTimelineCard.tsx` 신규. 최근 10건의 region cutover 이력 timeline (status: in-progress/completed/failed + actor + reason + 시각).
- **2 hook** — `useReplicas` · `useAuditChainHeadSHA` · `useFailoverHistory` (`web/src/api/hooks.ts`).
- **i18n** — `regions.*` namespace 40+ 키 추가 (`web/src/i18n/dict.ts` ko + en).

### `GET /api/v1/replication/failovers` backend endpoint (Stage 10.A-4)

`internal/api/handlers/replication.go::ListFailovers` 신규. admin 권한(`RequirePermission(ResourceTenantAdmin, ActionAdmin)`). query param `limit`(default 50, max 200) — initiated_at DESC 정렬. status는 completed_at NULL 여부로 도출(in-progress / completed). RegionTimelineCard fetch 진입점.

### Prometheus alert rule + ops runbook §13 (Stage 10.A-5)

- `deploy/prometheus/alerts/multi-region.yml` 신규 — 5 rule (`RosshieldReplicationLagWarning`/`Critical` · `RosshieldAuditChainHeadSeqMismatch` · `RosshieldHARoleSwap` · `RosshieldHAFailoverStorm`).
- `deploy/prometheus/alertmanager-sample.yml` 신규 — critical → PagerDuty + Slack #incident-response · warning → Slack #ops-alerts · info → Slack #audit-trail.
- `docs/operations/multi-region-failover-runbook.md` §13 — alert rule 매트릭스 + §13.1~§13.5 절차 (runbook_url annotation deep-link).

### e2e (Stage 10.A-6)

- `internal/platform/storage/postgres/regions_integration_test.go` 신규 (build tag `integration`) — `TestMultiRegionFailoverEndToEnd`. 2-region PG container fixture 재사용 + initial role + heartbeat + failover trigger + role swap + ListFailovers 등장 + audit chain head 진척 4 assertion. PG `LastInsertId` 한계 사실 명시(ListFailovers로 id 우회).
- `web/playwright/tests/regions.spec.ts` 신규 — admin login → nav '리전' click → `/regions` 도달 + PageHeader + AuditConsistencyCard + RegionTimelineCard render 검증. replicas 미시드 dev 환경에서도 페이지 본체는 mount(empty state 정상).

### design doc (Stage 10.A-1)

`docs/design/notes/phase10-backlog-design.md` — Phase 10 backlog 8 후보 매트릭스 + Top 3 권장 (A multi-region UI / D audit key rotation / E ros2-humble pack) + Stage 분해 + 결정 항목 D-P10-1·2.

## Migration Notes

### Customer 영향

- v0.8.5 customer는 v0.9.0 binary update 후 admin 권한 사용자가 `/regions` 페이지 접근 가능. 권한 없는 사용자에게는 라우터 가드(`requirePermission('tenant_admin', 'admin')`)가 차단.
- replication 미구성(single-region desktop·on-prem) 환경에서 `/regions` 진입 시 RegionHealthCard 영역이 empty state로 표시 — 비활성 상태에서도 페이지 본체 동작 정상.

### Prometheus / Alertmanager

- 신규 alert rule 활성화는 사용자 옵트인 — `deploy/prometheus/alerts/multi-region.yml`을 Prometheus rule_files에 명시 등록 시 동작. 기존 사용자 영향 0.
- runbook deep-link(`runbook_url` annotation) 활용은 PagerDuty/Slack notification 측 receiver 측 옵션.
