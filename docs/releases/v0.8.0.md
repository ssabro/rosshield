### Phase 9.3 — patroni.RoleProvider 구현 (`9975c3b`)

`internal/platform/ha/patroni/role_provider.go` (180줄) + `role_provider_test.go` (300줄, 12 test).

```go
// audit.RoleProvider + lagmetric.RoleProvider + cronsched.RoleProvider 3 interface 모두 duck-typed 자동 만족
type RoleProvider struct {
    leader atomic.Bool   // race-safe
    epoch  atomic.Int64  // Patroni timeline
}

func (rp *RoleProvider) IsLeader() bool       { return rp.leader.Load() }
func (rp *RoleProvider) CurrentEpoch() int64  { return rp.epoch.Load() }
```

#### 동작
1. `Start(ctx)` ticker goroutine (1s default)
2. GET `<PatroniURL>/cluster` (3s timeout)
3. JSON parse → `clusterResponse{Members, Leader, Timeline}`
4. `resolveLeader`: `Leader` field 우선 + `members[].role=="master"|"primary"` fallback
5. `leader == LocalHostname`이면 IsLeader=true, Timeline → epoch
6. HTTP/network/JSON error는 `logger.Warn` (panic 0 + 직전 atomic 값 보존)

#### Test (12)
- New validation 4 (URL · Hostname · default Interval/Timeout · trailing slash trim)
- pollOnce 5 (leader match · follower mismatch · members fallback · HTTP 500 graceful · network error graceful)
- Start/Close lifecycle 1 (200ms 동안 callCount + 2초 cleanup)
- resolveLeader 4 (Leader 우선 · master fallback · primary alias · empty fallback)

### Phase 9.4 — bootstrap `--ha-rp` flag + RoleProvider swap (`aa512d4`)

#### Config 5 신규 필드
```go
type Config struct {
    // ...
    HARP                    string        // "e25"|"patroni" — default "e25"
    PatroniURL              string        // 예: http://patroni:8008
    PatroniLocalHostname    string        // 본 Pod name (HOSTNAME env fallback)
    PatroniPollInterval     time.Duration // 0이면 1s
    PatroniRequestTimeout   time.Duration // 0이면 3s
}
```

#### CLI flag 5
```bash
--ha-rp=patroni
--ha-patroni-url=http://patroni:8008
--ha-patroni-local-hostname=$(hostname)  # 또는 HOSTNAME env
--ha-patroni-poll-interval=1s
--ha-patroni-request-timeout=3s
```

env: `ROSSHIELD_HA_RP` · `_PATRONI_URL` · `_PATRONI_LOCAL_HOSTNAME` (Kubernetes HOSTNAME fallback)

#### bootstrap 분기

```go
switch cfg.HARP {
case "patroni":  // Phase 9
    patroniRP, _ := patroni.New(Deps{...})
    patroniRP.Start(ctx)
    auditSvc.SetRoleProvider(patroniRP)
    sch.SetRoleProvider(patroniRP)
    lagCollector.SetRoleProvider(patroniRP)
    // ha.Manager 미사용 (platform.HA = nil)

case "", "e25":  // default, air-gap
    haMgr, _ := buildHAManager(cfg, store, logger)
    platform.HA = haMgr
    auditSvc.SetRoleProvider(haMgr)
    sch.SetRoleProvider(haMgr)
    lagCollector.SetRoleProvider(haMgr)
    haMgr.Start(ctx)
}
```

## Migration Notes

### 기존 customer (air-gap · single PG · 기존 E25)

영향 0. `--ha-rp=e25` (default)로 기존 E25 PG advisory lock 동작 유지.

### Kubernetes customer — Patroni 전환

1. **사전 준비**: Patroni Helm chart 배포 ([deploy/k8s/patroni/](../../deploy/k8s/patroni/) 참조)
2. **rosshield-server 환경 변수 변경**:
   ```yaml
   env:
     - name: ROSSHIELD_HA_ENABLED
       value: "1"
     - name: ROSSHIELD_HA_RP
       value: "patroni"
     - name: ROSSHIELD_HA_PATRONI_URL
       value: "http://rosshield-patroni-postgresql-ha-postgresql:8008"
   ```
3. **HOSTNAME env**: Kubernetes downward API가 자동 설정 — Pod name이 그대로 사용됨
4. **롤백**: `ROSSHIELD_HA_RP=e25`로 환경 변수만 변경하면 즉시 E25 로 복귀
