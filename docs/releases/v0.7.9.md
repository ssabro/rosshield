## Added

### MR.T6 application integration (`04dfb56`)

Phase 8 carryover "MR.T6 fence token enforcement application integration" 일소. v0.7.5의 MR.T6 schema test가 PG layer만 검증했던 한계를 application 통합 layer까지 확장.

#### 신규 mock `fakeAuditRole`

```go
type fakeAuditRole struct {
    leader bool
    epoch  int64
}
func (r *fakeAuditRole) IsLeader() bool       { return r.leader }
func (r *fakeAuditRole) CurrentEpoch() int64  { return r.epoch }
```

testcontainers fixture에서 audit.Service의 Deps.Role을 mock 주입.

#### TestAuditFenceEpochPropagatesCrossRegion

```
1. fixture primary에 audit.Service (sqliterepo.New, Role=leader epoch=42)
2. tenant 시드 + audit.Service.Append → audit_entries.leader_epoch=42 자동 저장
3. replication 5초 대기 후 standby에서 SELECT leader_epoch → 42 cross-region 일치 검증
```

**R5 cross-region fence token propagation 검증** — split-brain 방어 base 보장.

#### TestAuditFollowerRejectsAppend

```
1. audit.Service (Role=follower IsLeader=false)
2. Append 시도 → audit.ErrNotLeader 반환 검증
```

**HA single-writer 보장** — follower instance가 region-local PG에 write 시도해도 application layer가 차단.
