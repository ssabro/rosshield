### Audit rotation Stage 5 — verify CLI 확장 + prev_segment_hash chain (`6c7ab60`)

design doc Stage 5 — segment 간 chain 강화 + 외부 검증 CLI.

#### 마이그레이션 0035 (PG + sqlite)
- `audit_rotation_segments.prev_segment_hash BYTEA` — 직전 segment의 hash (chain link)
- 첫 segment(`segment_number=1`)는 NULL
- N+1 segment의 manifest.json에 prev_segment_hash 포함 (chain 외부 검증 가능)

#### builder/archiver/rotation 갱신
- `builder.go`: `Segment.PrevHash` field + `BuildSegmentWithPrev(prev []byte)` (BuildSegment는 wrapper 호환)
- `archiver.go`: manifest version `"1"` → `"2"`, `prevSegmentHash` 직렬화
- `rotation.go`: `Rotator.Rotate`가 `segmentNumber > 1`이면 직전 segment 조회 후 prev 자동 set + DB INSERT + chain gap 차단

#### verify CLI `rosshield-audit-verify`
신규 서브커맨드:
```bash
# 단일 segment verify
rosshield-audit-verify rotation \
  --archive-uri file:///var/rosshield/audit-archives/seg-0005.tar.gz \
  --expected-sha256 abc... \
  --expected-segment-hash def... \
  [--prev-segment-hash xyz...] \
  [--format json|table]

# batch chain verify (segment N → N+1 자동 forward)
rosshield-audit-verify rotation chain \
  --backend file:///var/rosshield/audit-archives/ \
  --from-segment 1 --to-segment 10
```

5 검증 단계: fetch · archiveSha256 · extract · segmentHash 재계산 · prevChain.

exit code: 0 PASS / 1 FAIL / 2 ARG.

#### 신규 test 15+
- 정상 verify PASS
- 변조 archive (sha256 mismatch) → FAIL
- 변조 entries (segment_hash mismatch) → FAIL
- prev_segment_hash mismatch → FAIL
- chain batch verify PASS
