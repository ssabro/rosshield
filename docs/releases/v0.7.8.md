## Added

### lagmetric HA leader-only gate (`99c8257`)

v0.7.6에서 도입한 `rosshield_replication_lag_seconds` collector가 HA cluster의 follower instance에서도 polling 수행해 metric 중복 emit + DB 부하 증가 가능성. cronsched·audit·rotationjob과 동일 RoleProvider 패턴으로 자동 gate.

#### Collector 확장 (`internal/platform/replication/lagmetric/collector.go`)

```go
// 신규 interface — ha.Manager 자동 만족
type RoleProvider interface {
    IsLeader() bool
}

// Deps 옵션 + lazy 주입
type Deps struct {
    // ... 기존 ...
    Role RoleProvider  // nil이면 single-instance 가정
}

// lazy 주입 (cronsched 패턴 일관)
func (c *Collector) SetRoleProvider(rp RoleProvider)
```

#### pollOnce 동작

```go
func (c *Collector) pollOnce(ctx context.Context) {
    if role := c.currentRole(); role != nil && !role.IsLeader() {
        c.deps.Registry.ReplicationLagSeconds.Reset()
        return  // Querier 호출 0회
    }
    // ... 기존 polling ...
}
```

- follower: metric Reset + Querier skip (DB 부하 0)
- leader: 기존 polling 수행
- nil Role: single-instance 가정 (모든 polling 수행)

#### race-safety

`atomic.Value`로 role 보호 — pollOnce goroutine과 SetRoleProvider 동시 호출 안전. `nilRoleSentinel` pattern으로 nil interface store 제약 회피.

#### bootstrap 결선

HA Manager 결선 직후 `lagCollector.SetRoleProvider(haMgr)` 호출 (sch·audit과 동일 lazy 패턴):

```go
sch.SetRoleProvider(haMgr)
if lagCollector != nil {
    lagCollector.SetRoleProvider(haMgr)
}
```

#### Test (4 신규)

| Test | 검증 |
|---|---|
| `TestPollOnce_FollowerSkipsQuery` | Role=follower면 Querier.Query 호출 0회 |
| `TestPollOnce_LeaderRunsQuery` | Role=leader면 정상 polling + metric Set |
| `TestPollOnce_FollowerResetsExistingMetric` | leader→follower 전환 시 metric 정리 |
| `TestPollOnce_NilRoleAlwaysPolls` | Role nil은 single-instance 가정 |

`fakeRole` mock + `fakeQuerier.calls` counter로 동작 검증.

## Migration Notes

### HA cluster customer

`--ha-enabled` + `--replication-lag-metric-enabled` 조합 활성화 시 자동 적용 — 별 설정 불필요. 부팅 후 leader instance만 `rosshield_replication_lag_seconds` emit, follower는 metric 0 라인.

### 빌드

```bash
git fetch --tags && git checkout v0.7.8
go build -o bin/rosshield-server ./cmd/rosshield-server
```

코드 동작:
- HA 비활성 (`HAEnabled=false`): single-instance 가정, 항상 polling — v0.7.6 동작 유지
- HA 활성: 본 release에서 leader gate 자동 적용
