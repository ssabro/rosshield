## Breaking Changes

**없음**. 본 release는 zero-value 호환 패턴을 일관 적용해 v0.2.0 client·script가 그대로 동작합니다.

- 신규 컬럼은 NULL/zero-value 허용 (`user_roles.source` default `'manual'` 등)
- 신규 endpoint는 추가형 (기존 endpoint 시그니처 0변경)
- RBAC middleware는 `RequireRole` 잔류 + `RequirePermission` 병행 (단계적 전환)

## Migration Notes

### 자동 마이그레이션 (운영자 액션 0)
서버 시작 시 `pressly/goose`가 다음 4건을 자동 적용합니다 (SQLite + PG 양쪽):

| 마이그레이션 | 내용 | 회수 가능 |
|---|---|---|
| `0027_robot_host_keys` | TOFU host key 저장 — scanrun SSH 통합 | ✅ (down sql) |
| `0028_user_roles_scope` | `scope_type`/`scope_id` 컬럼 추가 — fleet-scope RBAC | ✅ |
| `0029_sso_group_mappings` | SSO group → role 매핑 — RBAC fleet 정밀화 Stage 4 | ✅ |
| `0030_customer_intakes` | customer intake 도메인 — Phase 6 후보 1 R1 | ✅ |

### Backward-compat 노트
- **`AssignRoleScoped(... source)` 매개변수 추가**: 기존 호출부는 `source=""` 또는 `source="manual"` 전달 시 동작 보존. SSO 자동 동기화 경로만 `source="sso:<provider>"` 사용.
- **`RequireRole` → `RequirePermission` 단계적 전환**: 24 mutation gate 교체 완료, 잔여 admin-only gate(SSO·Invitation·Webhook mutation)는 `RequireRole` 유지.
- **JWT bindings claim 추가**: 신규 claim은 옵션 — 미포함 토큰도 시스템 role fallback으로 동작.

## Upgrade 절차 (v0.2.0 → v0.3.0)

### Docker Compose
```bash
# 1. 백업 (운영 환경 권장)
docker exec rosshield-server /usr/local/bin/rosshield backup create

# 2. 이미지 갱신
docker compose pull
docker compose up -d

# 3. 마이그레이션 자동 적용 (서버 부팅 시)
docker logs rosshield-server | grep "migration applied"
```

### Ubuntu Core snap
```bash
sudo snap refresh rosshield --channel=stable
# A/B OTA — post-refresh hook + 자동 rollback (E35) 보장
```

### HA cluster
- `docs/operations/ha-deployment.md` §5 무중단 rolling upgrade 절차 참조
- leader epoch fence token (마이그레이션 0022) 자동 갱신 — split-brain 방지
