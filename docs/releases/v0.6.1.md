## Added

### D-UI-2 List + Create/Detail Dialog 패턴

사용자 명시 요청 ("스캔 페이지에서 새 스캔 시작 다이얼로그가 기본 페이지에 왜 포함되어 있는지 모르겠어. 기본적으로 최근 세션 테이블을 표시하고, 우측 상단에 '새 스캔' 버튼을 두어 버튼을 누르면 다이얼로그가 표시되면서 스캔 시작이 되면 좋을 것 같고, 최근 세션 목록을 클릭 했을 때 상세정보도 다이얼로그 형식이 되었으면 좋겠어. 그리고 최근 세션 목록을 테이블로 표시할 때 긴 세션 ID, fleet ID를 굳이 풀로 표시할 필요가 있을까? ... 다른 페이지도 적용될 수 있으면 생각해서 적용해줘").

#### 신규 공유 컴포넌트
- **`TruncatedId`** — 긴 ULID/UUID → `prefix + … + suffix` 형식 (예: `scs_…HE0E`) + tooltip full ID + clipboard copy button + i18n toast

#### 페이지별 적용 (5 페이지)

| 페이지 | Create Dialog | Detail Dialog | TruncatedId 컬럼 |
|---|---|---|---|
| **scans** (`fa2c039`) | ✅ `CreateScanDialog` (form 분리) | ✅ `SessionDetailDialog` (live progress + cancel, `?session=` query param) | sessionId · fleetId · packId |
| **robots** (`fa2c039`) | ✅ `CreateRobotDialog` (RHF+zod) | ❌ `/robots/:robotId` drill-down 유지 (이력·credential 분량) | robotId · fleetId |
| **fleets** (`0e46737`) | ✅ `CreateFleetDialog` | ❌ `/fleets/:fleetId` drill-down 유지 | fleetId |
| **integrations** (`0e46737`) | ✅ `CreateEndpointDialog` (webhook RHF+zod) | ⏭ delivery detail (carryover) | endpoint.id · delivery.id |
| **users** (`0e46737`) | ✅ `CreateInvitationDialog` | ⏭ (사용자 단순 — popover 불필요) | invitation.id · invitedBy |

**핵심 결정**:
- **SessionDetailDialog URL state** — `?session=<id>` query param (open/close가 navigate로 통일, deep link + reload + 다른 탭 공유 호환)
- **robots/:robotId · fleets/:fleetId drill-down 페이지 유지** — 분량 큰 detail은 dialog 부적합
- **TruncatedId stub vs 정식판** — A sub-agent 정식판 + B sub-agent stub, cherry-pick 통합 시 정식판 채택
- **dict ko 일관성** — B sub-agent "Fleet 생성" → "플릿 생성" 통합 시 한국어 우선
