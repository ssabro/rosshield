## Added

### sqlite hot GC marker mode (`ee5f3c8`)

기존 0034가 PG GUC(`SET LOCAL rosshield.audit_gc_mode = 'on'`)로 `audit_entries_block_delete` trigger를 conditional 우회했지만 sqlite는 SET LOCAL/GUC 미지원. 본 release에서 sqlite-호환 marker mechanism 도입.

#### 마이그레이션 0036_audit_gc_marker

**sqlite**:
- `audit_gc_mode (id INTEGER PK CHECK id=1, active INT CHECK in 0,1)` table 신규
- `audit_entries_no_delete` trigger DROP/RECREATE with WHEN 절:
  ```sql
  WHEN NOT EXISTS (SELECT 1 FROM audit_gc_mode WHERE id=1 AND active=1)
  ```
- marker active=1일 때만 trigger silent pass, 그 외엔 `RAISE(ABORT, 'audit log is immutable')` 유지

**PG**: noop (이미 0034 GUC 사용). sequence 일관성 유지.

#### `rotation.HotGC` marker 분기

`HotGCDeps.UseMarkerMode` 신규 (bool). Run() 진입 시 driver 분기:

- **useMarkerMode=true** (sqlite): `INSERT OR REPLACE INTO audit_gc_mode VALUES (1, 1)` + defer `DELETE FROM audit_gc_mode WHERE id=1` (Tx COMMIT 전 자동 cleanup, ROLLBACK 시 자동 rollback)
- **useMarkerMode=false** (PG, default): 기존 `SET LOCAL rosshield.audit_gc_mode = 'on'` (PG GUC, Tx 끝에서 자동 reset)

#### bootstrap 자동 wiring

`cfg.StorageDriver == "sqlite"` 또는 빈 값(default sqlite) → UseMarkerMode=true 자동 주입. `Platform.HotGC` + `handlers.Deps.HotGC` 결선:

- 이전: `POST /api/v1/audit/gc/run`이 deps.HotGC nil로 503 응답
- v0.7.1: sqlite·PG 양 환경 모두 200 동작 (실 DELETE + audit.gc.complete emit + Head Seq 증가)

#### Test (3건)

- `TestHotGC_SetLocalRejectedOnSQLite` — 기존 보호막 유지 (UseMarkerMode=false default + sqlite SET LOCAL 거부 검증, PG 환경 misconfiguration 방어)
- `TestHotGC_SQLiteMarkerModeSucceeds` (신규) — UseMarkerMode=true에서 실제 DELETE 4 row + audit.gc.complete emit + Head Seq 증가
- `TestHotGC_SQLiteMarkerModeProtectsAgainstDirectDelete` (신규) — HotGC 완료 후 별 Tx에서 직접 `DELETE FROM audit_entries` 시도 → trigger RAISE(ABORT)

## Migration Notes

### 기존 sqlite customer

마이그레이션 0036은 부팅 시 자동 적용. 기존 `audit_entries_no_delete` trigger가 marker WHEN 절 버전으로 replace되지만, marker가 비활성 상태(default)에서 동작은 동일 — application code의 우발적 DELETE는 여전히 차단.

운영자는 hot GC를 활성화하려면:
- **manual 1회**: `POST /api/v1/audit/gc/run?dry_run=true` (admin token + ResourceTenantAdmin 권한)
- **정기 cron**: bootstrap에 `--audit-rotation-schedule '@every 720h'` 등 설정

### 기존 PG customer

영향 0. 0036 마이그레이션 noop, HotGC가 기존 SET LOCAL 경로 그대로 사용.
