## Added

### D-UI-1 Stage 5b drill-down a11y axe scan (`af5bfd1`)

D-UI-1 Stage 5b carryover C5b-3 일소. admin/auditor/인증 전/drill-down 거의 전체 a11y cover 도달.

#### `web/src/routes/__tests__/a11y-drilldown.test.tsx` (신규, 370+ 줄)

vitest + jsdom + vitest-axe. 10 페이지 light + 3 dark mode sampling.

| 페이지 | mount | mock |
|---|---|---|
| /audit | AuditPage | useAuditHead + useAuditCheckpoint |
| /reports | ReportsPage | useReports + useVerifyReport |
| /sso | SSOPage | useSSOProviders + CRUD |
| /integrations | IntegrationsPage | useWebhookEndpoints + summarizeDeliveries + Endpoint hooks |
| /license | LicensePage | useLicenseInfo |
| /advisor | AdvisorPage | useAdvisorConversations + useAdvisorConversation |
| /fleets/$fleetId | FleetDetailView({fleetId}) | useFleet + useRobots |
| /robots/$robotId | RobotDetailView({robotId}) | useRobot + useRobotResults + useDeleteRobot + useRotateCredential |
| /packs/$packKey | PackDetailPage (createFileRoute mock useParams) | usePack |
| /packs/$packKey/checks/$checkId | CheckDetailView({packKey, checkId}) | useCheck + useCheckSelftest |

dark mode sampling 3: SSO (form-heavy) + Fleet detail (table + meta) + Check detail (selftest list).

#### 페이지 named export 정비 (10 파일)

- audit/reports/sso/integrations/license/advisor/packs.$packKey: `function XPage` → `export function XPage`
- fleets.$fleetId/robots.$robotId/packs.$packKey.checks.$checkId: Route.useParams 분리 → `XDetailView({...params})` named export + thin wrapper

#### test 인프라 개선

- `createFileRoute` mock 반환에 `useParams` 포함 (PackDetailPage 같이 자체 Route.useParams 호출하는 페이지 cover)
- Link mock에 aria-label fallback (children empty 시 link-name violation 회피, production Link 영향 0)
