## Cloudflare vs Route53 비교

| 측면 | Cloudflare | Route53 |
|---|---|---|
| Global anycast | ✅ 200+ POP | 부분 |
| DDoS protection | ✅ 무료 (proxy on) | 별 WAF ($) |
| TTL minimum | 30s | 60s |
| AWS Cloud 통합 | 없음 | ✅ |
| 월 비용 | $5 Pro + LB ($5+$0.5/check) | usage-based |
| Monitor interval | 60s+ (Pro) | 30s+ |

**Cloudflare 권장**: Cloudflare proxy 이미 사용 + DDoS 보호 필요 + AWS 외 multi-cloud 환경.
**Route53 권장**: AWS multi-region 가정 + IaC 표준.

## Migration Notes

### Cloudflare 적용 절차

```bash
# 1. Cloudflare Pro plan 가입 + Load Balancer 옵션
# 2. API token 생성 (Zone:DNS:Edit, Zone:Load Balancer:Edit, Account:Load Balancer:Edit)
export CLOUDFLARE_API_TOKEN=<your-api-token>

cd deploy/terraform/multi-region-ha-cloudflare
cp envs/example.tfvars envs/prod.tfvars
# prod.tfvars 편집 (account_id·zone_id·hostname·endpoint)

terraform init
terraform plan -var-file=envs/prod.tfvars
terraform apply -var-file=envs/prod.tfvars
```

git fetch --tags && git checkout v0.7.7
