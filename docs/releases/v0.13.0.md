## Added

### `canonicalMetaJSONv3` + `ComputeEntryHashV3` (Stage 11.C-2)

`internal/domain/audit/hash.go` 갱신:

- `canonicalMetaJSONv3` 신규 — **9 키 알파벳순** (`action` < `actor` < `keyEpoch` < `leaderEpoch` < `occurredAt` < `outcome` < `seq` < `target` < `tenantId`). `keyEpoch` + `leaderEpoch` 는 `*int64 omitempty` — nil 이면 미노출 (v1 entry 와 byte-identical hash 보장하는 backward compat 안전망).
- `ComputeEntryHashV3(prevHash, payloadDigest Hash, e Entry) (Hash, error)` 신규 — signature는 v1 ComputeEntryHash 와 동일, 내부에서 canonicalMetaJSONv3 호출.
- `HashVersion` enum (V1=1, V3=3) + 내부 `computeEntryHash` dispatcher (도메인 내부 사용, 외부 노출 0).
- `canonicalMetaJSONv1` (기존 함수 rename, backward compat 엄격) — 기존 `ComputeEntryHash` signature + 동작 변경 0.
- 단위 test 신규: (a) nil-epoch byte-identical (v1 hash == v3 hash for v1 entries), (b) epoch set 시 v3 ≠ v1, (c) value 변경 민감도 (epoch 값 변경 시 hash mismatch), (d) determinism (같은 entry 두 번 직렬화 시 byte-identical), (e) 9 키 field mutation, (f) wire-format 알파벳순 sanity, (g) sha256 wire-reconstruction cross-check.

→ 신규 ComputeEntryHashV3 는 인프라만 — 실 Append 분기는 Stage 11.C-3+4 (chain transition marker).

### `audit.chain.hash_version_changed` transition marker entry + Append v3 분기 + ExportV3 (Stage 11.C-3+4)

`internal/domain/audit/transition.go` + `internal/domain/audit/sqliterepo/repo_hash_version.go` 신규:

- `audit.ActionHashVersionChanged = "audit.chain.hash_version_changed"` 상수 + `HashVersionTransitionMeta` payload struct (`FromVersion` + `ToVersion`).
- `audit.EnsureHashVersionTransition(ctx, repo, tenantID, actor)` 신규 — idempotent emit + setter cache. 첫 호출 시 transition marker entry 1회 INSERT, 이후 호출은 noop.
- `audit.HashVersionLocator` + `audit.HashVersionTransitionSetter` interface 신규 — 도메인 경계 P5 일관 (Repo 가 두 interface 자동 만족).
- `cmd/rosshield-server/audit_hash_transition.go` 신규 + `bootstrap.go` 갱신 — `RegisterCheckpointJob` 직후 1 Tx 안 idempotent emit. 두 번째 부팅부터는 noop (audit chain 자체가 source of truth, 별 메타 테이블 0).
- `sqliterepo.Repo.transitionSeq atomic.Int64` cache + `SetTransitionSeq`/`GetTransitionSeq`/`FindTransitionSeq`/`recomputeHashForSeq` 4 메서드 신규.
- **Append 경로 hash version 분기** — `entry.Seq <= transitionSeq` (또는 transitionSeq=0) 면 v1 hash, 그 외 v3 hash. transition entry 자체는 v1 hash (chain link 연속성 보장).
- **Verify 경로** 동일 분기 — `recomputeHashForSeq` 가 Append 와 같은 분기 일관.
- `audit.BundleVersionV3 = "v3"` const + `audit.MarshalEntryLineV3` 신규 — v2 wire 와 **byte-identical 호환 엄격** (`LeaderEpoch` 는 v3 만 노출, v2 변종은 절대 미노출 유지).
- `ExportEntryLine.LeaderEpoch *int64 json:"leaderEpoch,omitempty"` 신규 필드 + `ExportSignatureLine.HashVersionTransitionAt *int64 json:"_hashVersionTransitionAt,omitempty"` 신규 필드.
- `audit.ChainExporter.ExportV3` interface 신규 + `Repo.ExportV3` 구현 — v2 super-set + `BundleVersion="v3"` + `_hashVersionTransitionAt` + 각 entry 의 `LeaderEpoch` 노출.
- 마이그레이션 0건 — `audit_entries.action` 컬럼 + `audit_entries.key_epoch` + `leader_epoch` + `audit_chain_keys` 모두 Phase 10.D 결선 (append-only 일관).
- 단위 test 9 신규 (`transition_test.go` 5 + `repo_test.go` 4) + PG integration test 1 신규 (`hash_version_pg_integration_test.go` — transition emit + ExportV3 wire 검증).
- metric 신규 — `rosshield_audit_chain_hash_version{tenant}` Gauge (현재 활성 hash version, 1 또는 3) + `rosshield_audit_chain_hash_version_transition_total{tenant}` Counter (transition 1회 emit).

### `fg-verify` v3 + v1/v2/v3 3-tier backward compat (Stage 11.C-5)

- `exportOutput.BundleVersion` enum 확장 — `"v1" | "v2" | "v3"`.

- `verifyHashVersionTransitionMarker` 신규 — `_hashVersionTransitionAt` 와 bundle 안 transition entry seq (action = `audit.chain.hash_version_changed`) 일관성 검증.

  - v3 entries (seq 4~5)

- 단위 test 5 신규:

  - `V3Bundle_PostTransitionEntryTampered` — v3 entry `PayloadDigest` 변조 → chain FAIL 정확.
  - `V3Bundle_BoundaryHashFunctionMismatch` — 모든 entry v1 hash 계산 + `transitionAt=3` 명시 → chain FAIL 정확 (hash 함수 분기 어긋남).

  - `V3Bundle_SignatureTampered` — v3 `_signature` 변조 → signature FAIL 정확.
- `TestdataFixtures_PASS` 에 v3 case 추가 (회귀 가드).
- 외부 빌드 일관 — stdlib + crypto/ed25519 + audit 도메인 only. P5 일관 (Stage 10.D-5 패턴 cargo cult).
- v3 hash 함수 reimplement 회피 — `audit.ComputeEntryHashV3` 재사용 (단일 source of truth, 9 키 알파벳순 + nil omit 보장).
