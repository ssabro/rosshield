  - `cc3-risk-assessment.md` — CC3.1~CC3.4 (4 sub-controls, 결선 ~2.5/4 ~60%, formal risk register/fraud risk 외부 트랙 ★).

  - `cc6-logical-physical-access.md` — CC6.1~CC6.8 (8 sub-controls, CC6.4 DC physical 외부 트랙 ★).

  - `cc8-change-management.md` — CC8.1 (1 sub-control).
  - `cc9-risk-mitigation.md` — CC9.1~CC9.2 (2 sub-controls, CC9.2 vendor inventory 외부 트랙 ★).

- **Additional Criteria (A1~A5, 19 sub-controls — D-P11B-2 전체 5 cover)**:
  - `a1-availability.md` — A1.1~A1.3 (3 sub-controls, A1.3 environmental 외부 트랙 ★).

### `auditor` read-only role (Stage 11.B-5, D-P11B-3)

신규 role `auditor` — `internal/platform/authz/permission_matrix.go` 매트릭스에 이미 결선 (RBAC fine-grained, Phase 5+). audit.read/verify/export + compliance.read/export + report.read/verify/export + system.read 보유, write 권한 일체 0 (read-only 엄격). 신규 role 정의 코드 0 (기존 매트릭스 재사용 — minimal cost).

### audit log export wizard (`POST /api/v1/compliance/export` + UI, Stage 11.B-5)

- **backend** — `internal/api/handlers/compliance_export.go` 신규. `RequirePermission(ResourceAudit, ActionExport)` 게이트 (admin + auditor 통과). body `{"format":"v1|v2","period":"<duration>","tenantId":"<id>"}` (optional defaults). 응답은 audit bundle tar.gz + 응답 헤더에 `X-Audit-Entry-Seq` (export event entry seq). `audit.ChainExporter` 도메인 interface 신규 (`sqliterepo.Repo` 자동 만족) + `handlers.Deps.AuditExporter` / `AuditChainKeys` / `AuditSigner` 옵트인 게이트.
- **UI** — `/compliance/export` route 신규 + `compliance.export.*` i18n (ko + en). admin + auditor visible (system.read 게이트, fleet-admin/operator/read-only 자연 hide).
- **export bundle** — 기존 `sqliterepo.Repo.ExportV2` (Stage 10.D-5) 재사용. keyRepo nil 시 v1 fallback byte-identical 호환.
- **emit** — `audit.compliance.export` event 동일 Tx emit (entry seq → 응답 헤더, 외부 감사인 추적 가능).
- **단위 test** — backend 503/401/400/200(v1+v2)/empty body default + audit head seq 검증. rbac_integration_test endpoint count 29 → 30 (audit.export 1건 추가). web vitest exportErrorMessage 분기 + downloadAuditBundle fetch/blob mock.

### effectiveness dashboard (`GET /api/v1/compliance/effectiveness` + UI, Stage 11.B-6)

- **backend** — `internal/api/handlers/compliance_effectiveness.go` 신규. `RequirePermission(ResourceAudit, ActionExport)` 게이트 (admin + auditor). read-only handler — audit emit 0.
- **control → audit event 매핑** — `internal/domain/compliance/soc2_mapping.go` 신규 (Go map, 416줄). **14 카테고리 × 40 sub-control × audit action**. CC1~CC9 + A1·A2·A5 cover (D-P11B-2 default 일관).
- **aggregation** — `audit.EffectivenessAggregator` interface 신규 + `audit/sqliterepo.CountActionsByWindows` 구현. read-heavy query: `audit_entries action IN (...) + occurred_at >= ?` 단일 query (tenant scope index 활용). 1d / 7d / 30d window 동시 집계.
- **UI** — `/compliance/effectiveness` route 신규 + cover% hero · 매트릭스 표 · gaps 카드 + drill-down. cover% rollup = covered=true sub-control / total × 100.
- **i18n** — `compliance.dashboard.*` ko + en (40 신규 키).
- **단위 test** — soc2_mapping_test.go 5 (14 카테고리 + 40 sub-control + dedup actions + window offsets + BuildEffectivenessDashboard) + effectiveness_test.go 3 (audit/sqliterepo: empty actions + no entries + 1d/7d/30d aggregate) + compliance_effectiveness_test.go 3 (handlers: 503/401/happy path + read-only 검증) + compliance.effectiveness.test.tsx 15 (pending/success/empty/401/403/503/fallback + helpers).

- **90일 운영 효과성 측정**: SOC2 Type II 정의상 외부 firm이 측정. 본 트리는 자체 측정 도구(effectiveness dashboard + soc2-controls pack)만 cover.
- **Security awareness training (CC1.4)**: internal HR 트랙 또는 외부 트레이닝 벤더(KnowBe4/SANS 등).
- **Penetration testing (CC9.x)**: third-party 보안 firm(PortSwigger/Bishop Fox/Trail of Bits 등) 위탁.
