## Fixed (중요)

### cosign sign-blob bundle을 stdout 대신 임시 파일 경유 (`5df41f9`)

#### 잠재 prod 버그

cosign 2.x의 `cosign sign-blob --bundle=- -`는 stdout에 두 가지를 혼재 출력:
- base64 ECDSA signature (`MEUC...`)
- bundle JSON

이전 코드는 stdout 전체를 bundle로 간주해 DB의 `audit_rotation_segments.cosign_bundle` BYTEA 컬럼에 저장 → 외부 감사인이 `cosign verify-blob` 호출 시 첫 글자 'M'에서 JSON parser 실패:

```
Error: invalid character 'M' looking for beginning of value
```

**v0.7.0 이후 cosign 활성화한 customer 모두 영향** — 본 v0.7.2 이전 생성된 bundle은 verify 불가. 본 release 후 신규 rotation부터 정상 bundle 저장. **본 e2e test가 없었다면 customer가 archive 검증 시 발견할 잠재 위험.**

#### 정정

`os.CreateTemp("", "rosshield-cosign-bundle-*.json")` + `--bundle=<tmpfile>` 패턴 (cosign docs 권장):
1. 임시 파일 생성 (Unix 0o600 perms)
2. cosign이 해당 파일에 bundle JSON 직접 작성
3. `os.ReadFile` + defer cleanup
4. stdout/stderr는 capture하되 무시

### Migration sequence test 0036 등록 (`53b19aa`)

v0.7.1 `0036_audit_gc_marker` 추가 후 누락된 두 test list 갱신:
- `TestNoUnexpectedMigrationFiles` (PG): expectedMigrations에 0036 추가
- `TestStorageMigrateIdempotent` (sqlite): wantVersion 35 → 36

## Added

### cosign keyless e2e wire 호환성 test (`73f7a94`)

v0.7.x carryover 마지막 코드 작업 항목 일소.

#### 신규 파일 `cosign_e2e_test.go`

build tag `cosign_e2e`. 검증 흐름:
1. `CosignSigner.Sign`으로 실 archive 서명 → Fulcio 인증서 + Rekor SET 포함 bundle 생성
2. `cosign verify-blob --bundle=... --certificate-identity-regexp=... --certificate-oidc-issuer=...` 외부 호출 → "Verified OK" 검증
3. bundle 1 byte 변조 → verify-blob non-zero exit

`skipIfCosignUnavailable` 가드 — cosign binary 부재 또는 GitHub Actions OIDC env 부재 시 `t.Skip`.

#### 신규 CI job `cosign-e2e`

```yaml
permissions:
  id-token: write       # GitHub Actions OIDC token 활성
  contents: read
steps:
  - uses: sigstore/cosign-installer@v3
  - run: go test -tags=cosign_e2e -count=1 -timeout=8m ./internal/domain/audit/rotation/...
```

cosign이 `ACTIONS_ID_TOKEN_REQUEST_TOKEN`/`_URL` env를 자동 감지 → Fulcio에 keyless 인증서 발급 요청. OIDC issuer = `token.actions.githubusercontent.com`.

#### e2e 가치 입증

본 e2e test가 **release 직후 cosign 2.x 호환성 버그를 발견** — 단위 test (FakeSigner)는 binary 호출 안 해서 잡지 못한 wire layer 버그. customer가 archive 검증 시 발견할 잠재 위험을 release 전 차단.

## Migration Notes

### v0.7.0 ~ v0.7.1에서 cosign 활성화한 customer

본 release 이전 생성된 cosign bundle은 verify-blob 호환 안 됨 — bundle BYTEA 컬럼이 stdout 혼재 출력으로 손상된 상태일 수 있음. 해결책:

1. **v0.7.2 이후 신규 rotation은 자동 정상 bundle** — 별 조치 불필요.
2. **이전 rotation의 bundle 재서명** (옵션, 운영자 수동):
   - DB에서 `cosign_bundle = NULL`로 reset (cosign verify 시 skip)
   - 또는 archive를 base에 다시 서명 후 bundle column 직접 update

신규 customer (v0.7.2 첫 도입)는 영향 0.
