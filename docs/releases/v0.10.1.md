## Fixed

### depguard `audit-domain-isolation` 예외에 `keyrotationjob` 추가

v0.10.0 main CI Lint job:
```
internal/platform/scheduler/keyrotationjob/keyrotationjob.go:26:2: import 'github.com/ssabro/rosshield/internal/domain/audit/keyrotation' is not allowed from list 'audit-domain-isolation' (depguard)
```

**근본 원인**: depguard `audit-domain-isolation` 규칙은 도메인 격리(P5) 강제. 동일 패턴 `internal/platform/scheduler/rotationjob`은 entry-segment rotation 진입 시점에 예외 등록되었지만, Phase 10.D-3+4(`f7f045a`)에서 신규 `keyrotationjob`이 추가되며 같은 예외 등록을 빠뜨림.

**fix**: `.golangci.yml`에 `!**/internal/platform/scheduler/keyrotationjob/**` 패턴 추가. 두 어댑터 모두 cmd/* bootstrap이 KeyRotator/Rotator를 inject하는 동등 패턴(application service composer)이므로 예외 등록이 정합.

### gofmt -w 3 파일

v0.10.0 main CI Lint job:
```
cmd/rosshield-audit-verify/export_verify.go:290:1: File is not properly formatted (gofmt)
internal/domain/audit/export.go:59:1: File is not properly formatted (gofmt)
internal/domain/audit/sqliterepo/repo.go:363:1: File is not properly formatted (gofmt)
```

**근본 원인**: Stage 10.D-3+4 / 10.D-5 sub-agent 환경에서 commit 전 `gofmt -w` 적용 누락(memory `feedback_go_commit_pipeline.md` 정책 — local lint 없는 환경에서 commit 전 gofmt 필수).

**fix**: `gofmt -w` 3 파일 적용. 주석 indentation tab/space 정합.
