## Added

### Audit rotation Stage 4 hot GC (`4d42306`)

design doc Stage 4 — rotated segment 안의 audit_entries DELETE.

#### 마이그레이션 0034 — PG GUC trigger bypass
- `CREATE OR REPLACE FUNCTION audit_entries_block_delete()` — `current_setting('rosshield.audit_gc_mode', true) = 'on'`일 때만 RETURN OLD (DELETE 허용)
- 그 외 RAISE EXCEPTION (P9 불변성 유지)
- UPDATE는 별 함수 `audit_entries_block_update` 무변경
- sqlite: noop sequence holder (sqlite은 SET LOCAL 미지원, 별 epic carryover)

#### `internal/domain/audit/rotation/gc.go` (290줄)
```go
HotGCDeps{Policy RotationPolicy, Appender AuditAppender, Clock clock.Clock}
HotGCResult{DeletedCount, SegmentsProcessed []int64, OldestKeptEntrySeq, DryRun}
```
- `Run(ctx, tx, tenantID, dryRun)`:
  1. hot retention cutoff = now - HotRetention
  2. `ListSegmentsArchivedBefore(cutoff)` — archived 완료된 segment 식별
  3. `SET LOCAL rosshield.audit_gc_mode = 'on'`
  4. segment 범위 DELETE (tenant_id + seq 범위)
  5. MIN(seq) 조회 (oldest kept)
  6. `audit.gc.complete` entry emit (chain 정상 link)

#### `audit.gc.complete` entry — chain의 정상 link
canonical JSON payload: `{cutoffAt, deletedCount, oldestKeptEntrySeq, segmentNumbers}`. Target = `audit_chain:<tenantId>`.

#### Manual API + admin 권한
- `POST /api/v1/audit/gc/run?dry_run=true|false`
- 권한: `ResourceTenantAdmin, ActionAdmin` (다른 destructive 액션 일관)
- response: `{deletedCount, segmentsProcessed, oldestKeptEntrySeq, dryRun}`

#### Test (13 신규)
- 단위 7 (sqlite fake)
- PG integration 2 (build tag `integration` — 트리거 차단 + 우회 + DELETE + reset 실측)
- handler 4

### Audit rotation Stage 6 cron (`f431eb1`)

design doc Stage 6 — 자동 rotation job (in-process scheduler).

#### 신규 sub-package `internal/platform/scheduler/rotationjob/`
- cycle 회피 (audit/checkpoint.go가 scheduler interface import) — sub-package로 분리
- `Register(sch, deps, jobID, spec)` — spec="" no-op, deps nil → error
- `RunOnce(ctx, deps)` — tenant walk → 단일 Tx → `head.Seq` + `LatestSegmentNumber` + 직전 segment `LastEntryID` 기반 자동 산출
- `TenantLister` narrow interface + `TenantListerFunc` adapter

#### Bootstrap 결선
- `Config.AuditRotationSchedule string` (cron spec, "" = 비활성)
- `rotation.NewFileBackend(<DataDir>/audit-archives)` + `rotation.New` + `rotationjob.Register`
- HA gate: cronsched RoleProvider gate (E25 Stage 4a) 자동 적용 — follower silent skip
- graceful shutdown: 기존 `Platform.Shutdown`의 `p.Scheduler.Close(ctx)` 재사용

#### CLI flag / env
- `--audit-rotation-schedule` (flag) → `resolveAuditRotationSchedule` helper
- `ROSSHIELD_AUDIT_ROTATION_SCHEDULE` env fallback
- 권장 default: `@every 720h` (월 1회) 또는 `0 0 1 * *` (월 1일 자정)
