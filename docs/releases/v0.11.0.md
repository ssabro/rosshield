### C2 DDS topic ACL 4 check 깊이 확장 (Stage 10.E-5, jazzy + humble 양쪽 동기)

기존 `cmd_vel_acl_enforced` 1 topic 검증을 layered defense — 5 topic으로 확장. 양쪽 pack 동일 적용 (cargo cult):

- **`scan_acl_enforced`** — `/scan` (LiDAR) topic publisher count + `permissions.xml` ACL 매칭.
- **`odom_acl_enforced`** — `/odom` (Odometry) topic ACL 매칭 + nav stack frame_id 검증.
- **`tf_acl_enforced`** — `/tf` + `/tf_static` (Transform) topic ACL 매칭 (양 topic 통합 1 check).
- **`joint_states_acl_enforced`** — `/joint_states` (manipulator joint position) topic ACL 매칭.

D-P10E-3 결정 일관 — `permissions.xml` 파일 부재 또는 env `ROSSHIELD_SROS2_KEYSTORE` 미설정 시 PASS skip (false positive 회피, site policy 의존).

### C1 SROS2 cert chain 3 check 깊이 확장 (Stage 10.E-6, jazzy + humble 양쪽 동기)

기존 `sros2_keystore_exists` + `sros2_security_enable` 2 check를 cert chain 깊이로 확장. 양쪽 pack 동일 적용:

- **`sros2_cert_expiry`** — `$ROSSHIELD_SROS2_KEYSTORE/enclaves/*/cert.pem` 파일들의 `openssl x509 -checkend $(( 90 * 86400 ))` 검증. ≥90일 baseline (D-P10E-4 권장 default). expiry 임박 시 FAIL + rotation 지침 출력.
- **`sros2_ca_trust`** — `$ROSSHIELD_SROS2_KEYSTORE/ca.cert.pem`의 SHA-256 fingerprint를 env `ROSSHIELD_SROS2_CA_FINGERPRINT_SHA256` (사전 등록된 hex 64자)와 비교. env 미설정 시 PASS skip (site policy 의존).
- **`sros2_cert_revocation`** — `$ROSSHIELD_SROS2_KEYSTORE/ca.crl.pem`의 `Next Update` 필드가 ≥90일 미래임을 `openssl crl -nextupdate` 로 검증. CRL 파일 부재 시 PASS skip.

D-P10E-4 결정 일관 — 90일 baseline은 quarterly cert rotation cadence와 정합. 30일 또는 site-specific threshold 옵트인은 별 carryover.

### Jazzy customer 자동 확장

```bash
# 1. v0.11.0 binary update — ros2-jazzy 22 → 29 check 자동 확장.
git checkout v0.11.0
make build
make pack-archive          # 기존 jazzy archive 갱신.

# 2. 신규 check (DDS 4 + SROS2 3)는 env 미설정 시 PASS skip — 운영자 알람 없음.
# 3. (권장) SROS2 keystore 사전 등록 후 깊이 cover 활성.
export ROSSHIELD_SROS2_KEYSTORE=/opt/ros/keystore
export ROSSHIELD_SROS2_CA_FINGERPRINT_SHA256=<openssl x509 -in /opt/ros/keystore/ca.cert.pem -sha256 -fingerprint | cut -d= -f2 | tr -d :>
```

### SROS2 keystore env var 사전 등록 (권장)

신규 7 check는 site policy 의존 default — env 미설정 시 PASS skip (false positive 회피). 깊이 cover 활성을 원하면 customer 환경에 다음 env 사전 등록:

- `ROSSHIELD_SROS2_KEYSTORE` — SROS2 keystore 절대 경로 (default `$HOME/sros2_keystore`).
- `ROSSHIELD_SROS2_CA_FINGERPRINT_SHA256` — `ca.cert.pem` 의 SHA-256 fingerprint hex 64자 (대문자, `:` 없는 형태).
