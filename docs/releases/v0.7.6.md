## Added

### Phase 8 MR.T8 — replication lag Prometheus metric (`c834388`)

v0.7.5 한계 항목 "MR.T8 Prometheus metric emit" 일소.

#### Registry 확장 (`metrics.go`)

```go
type Registry struct {
    // ...
    ReplicationLagSeconds *prometheus.GaugeVec  // label: application_name
}
```

Help: `"PG logical replication lag in seconds, partitioned by subscriber application_name. Emitted only on primary instances when replication is enabled."`

#### 신규 패키지 `internal/platform/replication/lagmetric/`

- **`Querier` interface** — pgx Query 단일 메서드 (test mock 친화, `*pgxpool.Pool` 자동 만족)
- **`Collector`** — `New(Deps)`/`Start(ctx)`/`Close()`/`pollOnce(ctx)`:
  - 첫 polling 즉시 + ticker 30s 간격 (Interval 옵션)
  - `SELECT application_name, EXTRACT(EPOCH FROM replay_lag) FROM pg_stat_replication`
  - 매 polling 시작 시 `Gauge.Reset()` — stale label cleanup
  - query error는 `logger.Warn`만 — collector goroutine 죽지 않음

#### bootstrap 결선 (`bootstrap.go` + `main.go`)

조건: `ReplicationLagMetricEnabled=true` + `ReplicationConfig.Enabled=true` + `Role=primary` + PG storage. 그 외는 silent skip(sqlite/standby/disabled graceful).

CLI:
```bash
--replication-lag-metric-enabled
--replication-lag-metric-interval=30s
```

env: `ROSSHIELD_REPLICATION_LAG_METRIC_{ENABLED,INTERVAL}`.

#### Test

9 단위:
- `New` validation 4 (Querier 필수, Registry 필수, default Interval, explicit Interval)
- `pollOnce` 4 (정상 emit per subscriber, stale Reset, query error graceful, 빈 subscriber)
- `Start` lifecycle 1 (ctx cancel cleanup 2초 안)

## Fixed

### audit_entries TIMESTAMPTZ 호환 (`1ee429e`)

v0.7.5 commit `36fb8a4` 후속 PG integration CI fail (MR.T5·T6):
```
ERROR: column "occurred_at" is of type timestamp with time zone
but expression is of type text (SQLSTATE 42804)
```

마이그레이션 0024 (E22-F PG-native hotpath)가 `audit_entries.occurred_at` + `audit_chain_heads.updated_at`을 TEXT → TIMESTAMPTZ로 변경. 본 test의 `NOW()::TEXT` cast 거부.

**정정**: `NOW()::TEXT` → `NOW()` (TIMESTAMPTZ). tenants.created_at은 TEXT 유지(0024 영향 외) — 변경 없음.

### go.mod tidy drift (`1ee429e`)

`aws/smithy-go v1.25.1` indirect → direct 승격. v0.7.2 cosign middleware의 `smithymiddleware`/`smithyhttp` import 누락.

## Migration Notes

### lag metric 활성화 (옵션, primary PG만)

```bash
export ROSSHIELD_REPLICATION_ENABLED=1
export ROSSHIELD_REPLICATION_ROLE=primary
export ROSSHIELD_REPLICATION_LAG_METRIC_ENABLED=1
export ROSSHIELD_REPLICATION_LAG_METRIC_INTERVAL=30s
```

`/metrics` endpoint에서 다음 시계열 노출:
```
# HELP rosshield_replication_lag_seconds PG logical replication lag in seconds...
# TYPE rosshield_replication_lag_seconds gauge
rosshield_replication_lag_seconds{application_name="rosshield_main_sub"} 0.42
```

### Alertmanager rule 예시

```yaml
groups:
- name: rosshield-replication
  rules:
  - alert: ReplicationLagHigh
    expr: rosshield_replication_lag_seconds > 60
    for: 5m
    annotations:
      summary: "Replication lag > 60s — RPO at risk"
```

### 빌드

```bash
git fetch --tags && git checkout v0.7.6
cd web && pnpm install --frozen-lockfile && pnpm build
cd .. && go build -o bin/rosshield-server ./cmd/rosshield-server
```
