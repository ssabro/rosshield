# 00. 미션 및 포지셔닝

## 0.1 미션

> **ROS2 로봇 플릿이 정의된 보안 기준을 지속적으로 지키고 있음을, 감사인이 받아들이는 형태의 증거와 함께 증명한다.**

### 세 개의 핵심 동사

1. **증명한다** — 결과가 아니라 증거를 남긴다. 규제 대응이 최종 목적지.
2. **지속적으로** — 단발 감사가 아니라 드리프트 탐지·주기 실행·차분 스캔.
3. **받아들이는 형태** — 감사인·규제기관·고객사 보안팀이 신뢰할 수 있는 서명·해시 체인·제3자 검증 가능성.

## 0.2 한 문장 포지셔닝

> **"CAI가 로봇을 침투한다면, 우리는 플릿이 그 공격에 대비되어 있는지 매일 증명한다."**

## 0.3 시장 내 위치

```
                       자율성/공격성
                            ▲
                            │
     CAI (Alias Robotics)   │
     · 에이전트 기반          │
     · 공격/탐색              │
     · 비결정적 증거          │
  ───────────────────────────┼───────────────────────────▶  결정성/감사성
                            │
                            │   본 제품
                            │   · 체크리스트 기반
                            │   · 준수/증명
                            │   · 결정론적 증거
                            │   · 서명된 리포트
                            ▼
                       방어/감사 중심
```

- **좌상**(자율·공격): CAI, Metasploit/Cobalt Strike 계열
- **좌하**(자율·방어): SIEM·XDR (일반 IT 영역, 로봇 특화 아님)
- **우상**(체크·공격): OpenVAS·Nessus (네트워크 취약점 스캐너, 체크 기반이지만 공격적 탐지)
- **우하(본 제품)**: ROS2 플릿 전용 **체크리스트 감사 + 컴플라이언스 증명 + 드리프트·트렌드**

## 0.4 대상 고객 세그먼트

| 세그먼트 | 대표 고객 유형 | 구매 동인 | 선호 배포 형태 |
|---|---|---|---|
| **소규모 기술팀** | 로봇 스타트업, 대학 랩 | 펀딩/납품 시 보안 자료 필요 | 데스크톱 (Free/Pro) |
| **중규모 로봇 운영자** | AMR 물류, 병원 로봇 운영사 | ISMS-P/ISO27001 인증, 고객 요구 대응 | 온프렘 서버 |
| **대규모 제조업** | 자동차·전자 공장 AMR/AGV 운영 | 내부 감사·공급망 보안 실사 | 온프렘 서버 / 어플라이언스 |
| **초민감 인프라** | 방산·공항·원전·국방 | 법적 의무(주요정보통신기반시설 등) | 어플라이언스 이미지, 에어갭 |
| **로봇 제조사** | 자체 로봇 출하 전 보안 검증 | 출하 기준·OEM 계약 대응 | OEM 라이선스/온프렘 |
| **SI·컨설팅** | 대형 SI, 보안 컨설팅 | 고객 감사 대행 | 온프렘 or 리셀러 어플라이언스 |

## 0.5 구매자 vs 사용자 구분

- **구매 의사결정자**: CISO, 보안팀장, 내부감사, 컴플라이언스 담당 → "감사인·규제 대응 가능한가?"
- **일상 사용자**: 로봇 운영자, SRE, 보안 엔지니어 → "매일 돌리기 편한가? 드리프트 알림 정확한가?"
- **가치 소비자**: 경영진·고객사 → "한 장 리포트로 납득되는가?"

**세 층 모두에게 하나의 제품이 말을 걸어야 한다.** 이는 UI·문서·리포트 설계의 핵심 제약이다.

## 0.6 CAI와의 차별화 (냉정)

| 축 | CAI | 본 제품 |
|---|---|---|
| 패러다임 | LLM 에이전트 자율 실행 | 체크리스트 기반 결정론적 감사 |
| 주 목적 | 공격 가능성 발견·시연 | 방어 상태 지속 증명 |
| 증거 성격 | 에이전트 탐색 로그 (비결정적) | 원본 명령 출력 + 해시 체인 (결정론적) |
| UX | CLI + REPL | 웹 콘솔 + 데스크톱 + CLI |
| 배포 | `pip install` self-host | 데스크톱/온프렘/어플라이언스 3종 |
| 컴플라이언스 | IEC 62443 · ISO 10218 (탐색형) | CIS · NIST · ISO27001 · ISMS-P · 주요정보통신기반시설 (매핑형) |
| 언어 | 영어 중심 | 한국어 1급, 영어 병행 |
| LLM 의존도 | 필수 (300+ 모델) | 완전 옵트인, 오프라인 1급 |
| 에어갭 | 가능은 하나 보조 | 1급 설계 가정 |
| 수익 모델 | OSS + PRO 호스팅 | 구독 + 컨텐츠 팩 + 지원 SLA |

### 경쟁이 아니라 보완

CAI는 **"공격자 시점에서 뚫릴 수 있음을 증명"**합니다. 본 제품은 **"방어자 시점에서 매일 지켜지고 있음을 증명"**합니다. 성숙한 고객 조직은 **둘 다 필요**합니다. 우리가 CAI와 실제로 싸우는 영역은 "플릿 운영 대시보드·규제 매핑·감사 증거"이고, 이 영역은 CAI 로드맵 주력 밖입니다.

### 통합 가능성

향후 "옵셔널 공격 시뮬레이션" 기능이 필요하면 **CAI를 외부 plugin backend**로 호출하는 얇은 어댑터로 도입. 자체 에이전트 프레임워크는 만들지 않습니다.

## 0.7 가치 제안 (Value Proposition)

### 고객이 얻는 것

1. **"우리 로봇 플릿이 기준을 지키고 있다"는 매일 갱신되는 증거**
2. **규제·고객사 감사 대응 시 제출할 수 있는 서명된 리포트**
3. **변화(드리프트) 발생 시 24시간 내 알림**
4. **새 로봇 투입 시 온보딩 마법사로 즉시 감사 범위 편입**
5. **벤치마크·매핑 업데이트 구독으로 규제 변화에 자동 대응**

### 고객이 하지 않아도 되는 것

1. **보안 전문가 없이 체크리스트를 수작업으로 돌림**
2. **감사 때마다 증거를 재생산**
3. **리포트를 엑셀로 하나하나 꾸밈**
4. **규제 변경 사항을 직접 추적해 체크리스트 갱신**

## 0.8 초기 침투 전략

### 진입 쐐기(Wedge)

**"CIS Ubuntu 24.04 벤치마크를 ROS2 Jazzy 로봇 플릿에서 결과 + 서명된 리포트와 함께 돌려주는 가장 간단한 도구."**

복잡한 것 다 빼고 이 한 줄로 **PoC → 유료 전환**이 가능해야 합니다.

### 첫 고객 목표

1. 한국어 UX + ISMS-P·주요정보통신기반시설 매핑이 필요한 **국내 중대형 로봇 운영자** 2~3곳
2. 방산·공공 쪽 SI 파트너와 **어플라이언스 PoC** 1건
3. 로봇 제조사 1곳과 **OEM 라이선스 협의**

이 세 축이 Stage 1~2 기간(약 12~18개월) 초기 검증 지표입니다.

## 0.9 해외 진출

- **Phase 0**: 국내 집중. 영어 UI는 2급 (라벨 번역 + 기본 문서).
- **Phase 1**: 일본 · 동남아 제조업 (한국 SI 파트너의 해외 레퍼런스 타고).
- **Phase 2**: 유럽 (IEC 62443 · NIS2 대응), 북미 (NIST 800-53·CMMC).

영어 1급 투자는 국내 연 매출 일정 규모 돌파 이후 — 무리하게 앞당기지 않습니다.

## 0.10 이 문서의 결론

1. 시장 내 "**결정론적 감사 + 컴플라이언스 증명 + 로봇 특화**" 사분면은 **비어 있거나 약**합니다.
2. CAI는 경쟁자가 아니라 **보완재**로 포지셔닝합니다.
3. 성공의 열쇠는 **감사인이 받아들이는 증거**와 **지속 갱신되는 컨텐츠**이며, 이는 기술적 해자와 상업적 해자의 동시 구축으로 이어집니다.
